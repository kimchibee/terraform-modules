# 작업 계획서: 공동 모듈 AVM 전략 (이해도 확인용)

## 1. 목표 (고정)

- **공동 모듈 레포(`terraform-modules`)는 항상 사용**한다.
- **가능하면 AVM만**으로 스택 요구를 충족한다.
- **불가능한 경우**에만 공동 모듈 안에서 **`azurerm` 예외**를 둔다 (IaC에 로직을 흩어두지 않는 것을 원칙으로 한다).
- **IaC**는 `git::https://github.com/kimchibee/terraform-modules.git//terraform_modules/...?ref=...` 로 공동 모듈만 참조한다.

## 2. 판단 요약

| 구분 | 공동 모듈에 넣는 형태 |
|------|------------------------|
| Registry AVM으로 끝까지 커버 | **AVM-only 래퍼** (`module "avm"` + 버전 고정) |
| AVM + 소량 보강이 같은 배포 단위 | **AVM + `azurerm` 예외** (한 모듈 안) |
| 적합한 AVM 없음·계약 불일치 | **`azurerm`만** (예외 모듈로 명시) |

AVM 버전을 올릴 때 **같은 모듈 안의 예외 `azurerm`와 결합이 깨질 수 있으므로** 혼합 모듈은 **회귀 검증** 대상이다.

## 3. 본 문서의 범위

- 현재 레포의 모듈을 **위 목표에 맞게 분류**한다 (`docs/AVM_COVERAGE.md`).
- 코드 구조를 바꾸는 **대규모 리팩터는 본 계획서 범위 밖**이며, 분류·문서화를 우선한다.

## 4. 산출물

- `docs/AVM_COVERAGE.md` — 모듈별 AVM / 혼합 / azurerm-only 표
- (후속) 모듈별 `versions.tf`·AVM `version` 정합 유지, 태그로 `ref` 고정

## 5. 완료 기준

- 팀이 **“어떤 모듈이 순수 AVM이고, 어떤 모듈이 혼합/예외인지”** 한 문서로 동의할 수 있다.
